## 1. PI 에코시스템 개요

**Pi**(`@mariozechner/pi-coding-agent` v0.63.1)는 Mario Zechner가 만든 터미널 기반 AI 코딩 에이전트 CLI.
확장 시스템이 핵심 — npm 패키지로 도구, 스킬, 프롬프트 템플릿을 배포/설치.

```
pi install npm:<package-name>
pi install https://github.com/user/repo
```

**확장 포인트 4가지:**
| 타입 | 설명 | 경로 |
|------|------|------|
| Extensions | TypeScript 모듈, 런타임 로드 | `~/.pi/agent/extensions/*.ts` |
| Skills | 마크다운 지침 파일 | `~/.pi/agent/skills/*.md` |
| Prompt Templates | 프론트매터 + 프롬프트 | `.md` 파일, 슬래시 커맨드로 호출 |
| Themes | TUI 테마 | 설정 파일 |

## 2. 연구 대상별 분석

### 2.1 pi-web-access (v0.10.4)

> 소스: [nicobailon/pi-web-access](https://github.com/nicobailon/pi-web-access) · MIT

**핵심 기능**: 웹 검색 + 콘텐츠 추출 + YouTube/PDF/비디오 분석

**4가지 도구:**
| Tool | 용도 |
|------|------|
| `web_search` | 웹 검색 → 합성 답변 + 출처 |
| `code_search` | Exa MCP로 코드/API 레퍼런스 검색 |
| `fetch_content` | URL/GitHub/YouTube/PDF → 마크다운 추출 |
| `get_search_content` | 이전 세션 결과 조회 |

**검색 큐레이션 모드 (`summary-review`)**:
1. 검색 실행 → 결과 수집
2. 큐레이터 UI 오픈 — 사용자가 결과 선택/추가/편집
3. 자동 요약 초안 생성 (LLM 1회 호출)
4. 사용자 승인 또는 타임아웃(20초) 시 자동 제출
5. 승인된 요약이 에이전트에게 반환

**검색 프로바이더 체인:**
- `web_search`: Exa → Perplexity → Gemini API → Gemini Web
- YouTube: Gemini Web → Gemini API → Perplexity
- 차단된 페이지: Jina Reader → Gemini URL Context → Gemini Web

**Exa**: API 키 없이 MCP로 사용 가능 (zero-config)

**kittypaw 적용 포인트:**
- Http 프리미티브 확장 → 웹 검색 + 콘텐츠 추출 기능
- 검색 큐레이션 = 사용자가 검색 결과를 필터링한 뒤 스킬에 주입
- 프로바이더 폴백 체인 패턴

### 2.2 pi-prompt-template-model (v0.6.6)

> 소스: [nicobailon/pi-prompt-template-model](https://github.com/nicobailon/pi-prompt-template-model) · MIT

**핵심 기능**: 프롬프트 템플릿에 모델 선택, 루프, 로테이션, 서브에이전트 위임 추가

**모델 로테이션 예시:**
```yaml
---
model: claude-opus-4-6, gpt-5.4, gpt-5.3-codex
thinking: high, xhigh, off
loop: 9
rotate: true
converge: true   # 파일 변경 없으면 조기 종료
fresh: true       # 이터레이션 간 컨텍스트 압축
---
```

| 필드 | 역할 |
|------|------|
| `loop` | N회 반복 (또는 `true`/`"unlimited"`) |
| `rotate` | 이터레이션마다 다른 모델 사용 |
| `converge` | 변경 없으면 조기 종료 |
| `fresh` | 이터레이션 간 컨텍스트 리셋 |
| `subagent` | 서브에이전트로 위임 |
| `inheritContext` | 현재 대화를 서브에이전트에 포크 |
| `chain` | 순차/병렬 파이프라인 |

**서브에이전트 동작:**
- `subagent: true` → pi를 재귀적으로 subprocess로 실행
- `inheritContext: true` → 현재 대화 스냅샷을 포크
- `chainContext: "summary"` → 이전 작업을 요약해서 전달

**kittypaw 적용 포인트:**
- 스킬 실행 시 모델 선택 프론트매터 (`package.toml`에 `model` 필드)
- 복잡한 스킬은 여러 모델 체인으로 단계별 처리
- teach loop에서 모델 로테이션으로 다양한 코드 생성 → 최적 선택

### 2.3 pi-autoresearch

> 소스: [davebcn87/pi-autoresearch](https://github.com/davebcn87/pi-autoresearch)
> 설치: `pi install https://github.com/davebcn87/pi-autoresearch` (npm 미등록)

**핵심 기능**: 자율 최적화 루프 — 코드 수정 → 벤치마크 → 유지/리버트 → 반복

**루프 사이클:**
```
Edit → Commit → run_experiment → log_experiment → Keep or Revert → Repeat
```

**세션 재개 파일 2개:**
| 파일 | 역할 |
|------|------|
| `autoresearch.jsonl` | 실험 로그 (append-only). 커밋, 메트릭, 상태, 타임스탬프 |
| `autoresearch.md` | 진행 상황 문서. 목표, 시도한 전략, 막다른 길, 주요 성과 |

→ 새 에이전트 세션이 이 두 파일만으로 **완전한 컨텍스트 복원** 가능

**메트릭 출력 형식:**
```
METRIC name=value
```

**신뢰도 점수** (3회+ 실험 후):
```
Confidence = |best_improvement| / MAD(모든 메트릭)
```
- ≥2.0: 🟢 개선이 실제일 가능성 높음
- 1.0–2.0: 🟡 미미한 개선
- <1.0: 🔴 노이즈 범위 내

**정합성 검사** (`autoresearch.checks.sh`):
```bash
#!/bin/bash
set -euo pipefail
pnpm test --run
pnpm typecheck
```

**kittypaw 적용 포인트:**
- 스킬 자동 최적화 모드: "이 스킬의 응답 시간을 줄여줘" → 자동 루프
- `autoresearch.jsonl` 패턴 → 스킬 실행 로그 + 성능 추적
- 신뢰도 점수로 스킬 품질 자동 측정
- checks.sh 패턴 → 스킬 배포 전 자동 검증

### 2.4 model-router (umgbhalla/pi-config)

> 소스: [umgbhalla/pi-config/extensions/model-router.ts](https://github.com/umgbhalla/pi-config/blob/main/extensions/model-router.ts)

**핵심 기능**: 프롬프트 키워드로 최적 모델 자동 선택

**4가지 라우트:**
| Route | 모델 | Thinking | 키워드 예시 |
|-------|------|----------|------------|
| `hard_reasoning` | gpt-5.4 | high | debug, root cause, architecture, race condition |
| `fast_code` | gpt-5.3-codex-spark | low | quick fix, small change, patch, cleanup |
| `smart_small_refactor` | gpt-5.4-mini | medium | small refactor, improve readability, simplify |
| `frontend_design` | claude-opus-4-6 | medium | frontend, design, ui, ux, animation |

**2단계 신뢰도:**
- ≥2 키워드 매치 → `high` → 자동 전환
- =1 키워드 매치 → `medium` → 추천만 (자동 전환 안함)

**시스템 프롬프트 주입:**
```
[model-router] Auto-routed this turn to gpt-5.4 because the prompt matched hard_reasoning (debug, root cause).
```

**kittypaw 적용 포인트:**
- teach loop에서 사용자 설명 → 키워드 분류 → 모델 자동 선택
- 단순 스케줄링 스킬은 빠른 모델, 복잡한 통합 스킬은 추론 모델
- 2단계 신뢰도 게이팅 패턴 재활용
- `kittypaw.toml`에 `[model_routing]` 섹션 추가

### 2.6 AI 비서 워크플로우 (ginipigi 아티클)

> 출처: @ginipigi 트위터 아티클 (2026-03-28)

**핵심 패턴**: Claude Projects를 AI 비서로 세팅하는 실전 워크플로우

**지침 구조:**
1. 캐릭터 설정 (이름, 역할, 상징)
2. 입맛 맞추기 (싫어하는 것 위주로 명시)
3. 배경 지식 주입 (계정 성격, 팔로워 프로필)
4. 사실 확인 원칙 (추측 명시, 수치 금지)

**실전 워크플로우:**
- 매일: 팩트체크 (기사 → 트윗 초안 → AI 비판적 검토)
- 주간: 콘텐츠 성과 회고 (데이터 → 패턴 분석)
- 수시: 심층 분석 (기관 리포트 → 맥락 있는 분석)

**인간 게이트 원칙:**
- 최종 판단은 반드시 인간이
- 기획은 AI 켜기 전에 내 생각 먼저
- 수치는 무조건 교차검증
- 틀리면 빠르게 공개 정정

**kittypaw 적용 포인트:**
- "AI 비서 세팅 위자드" 스킬
- 팩트체크 파이프라인 스킬
- 콘텐츠 회고/분석 스킬
- 교차검증 자동화 (복수 LLM 호출 → 비교)

```
$ kittypaw teach "트위터에 올릴 경제 트윗 팩트체크해줘"
🐾 Teaching: 경제 트윗 팩트체크 스킬
🤖 Capabilities: Web.search, Llm (x2 for cross-check), Telegram

사용법:
  트윗 초안 입력 → 웹 검색으로 사실 확인 → 2개 LLM 교차검증
  → 오류 발견 시 정정 제안 + 출처 첨부
```
